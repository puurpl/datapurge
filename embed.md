# Embedding DataPurge

Add a data broker opt-out tool to your website. Your visitors can generate and send legally-backed deletion requests to 200+ data brokers without leaving your page.

All user data stays in the visitor's browser — nothing is sent to any server.

## Option 1: iframe Embed (Recommended)

Paste this into your HTML:

```html
<iframe
  src="https://datapurge.iamnottheproduct.com/embed.html"
  style="width: 100%; height: 700px; border: 1px solid #e2e8f0; border-radius: 12px;"
  title="DataPurge — Data Broker Opt-Out Tool"
  allow="clipboard-write"
  loading="lazy">
</iframe>
```

**That's it.** Works on any platform — WordPress, Squarespace, Wix, Webflow, Ghost, Hugo, Jekyll, raw HTML, anything that supports iframes.

### Adjusting the height

The embed adapts to its container width. Recommended heights:

| Context | Height |
|---------|--------|
| Full page embed | `700px` or `80vh` |
| Sidebar widget | `600px` |
| Blog post | `650px` |

### Platform-specific notes

**WordPress** — Use a Custom HTML block, paste the iframe code.

**Squarespace** — Add a Code Block, paste the iframe code. Works in all templates.

**Webflow** — Add an Embed element, paste the iframe code.

**Wix** — Add an HTML iframe element, paste the `src` URL.

**Ghost** — Use an HTML card, paste the iframe code.

**React / Next.js / Vue / Svelte** — Paste the iframe into a component. No special wrappers needed.

## Option 2: Self-Hosted

If you prefer to serve the tool from your own domain (for branding, compliance, or reliability), download the self-hosted file:

1. Download [`datapurge-embed.html`](https://datapurge.iamnottheproduct.com/datapurge-embed.html)
2. Place it on your web server
3. Link or iframe it from your pages

The self-hosted file loads CSS, JavaScript, and data from `datapurge.iamnottheproduct.com` — so it always uses the latest broker database and legal templates. You only host the HTML shell.

```html
<!-- Link directly to your hosted copy -->
<a href="/datapurge-embed.html" class="button">Opt Out of Data Brokers</a>

<!-- Or embed it in an iframe -->
<iframe
  src="/datapurge-embed.html"
  style="width: 100%; height: 700px; border: 1px solid #e2e8f0; border-radius: 12px;"
  title="DataPurge — Data Broker Opt-Out Tool"
  allow="clipboard-write"
  loading="lazy">
</iframe>
```

### Fully self-contained hosting

To host everything locally (no external requests):

```bash
git clone https://github.com/puurpl/datapurge.git
cd opt-out
pip install pyyaml && python3 build.py
# Serve the web/ directory from your web server
```

## Privacy & Security

- **PII never leaves the visitor's browser.** The tool uses `localStorage` for data. In iframe mode, localStorage is scoped to the DataPurge origin — the host page cannot access it.
- **No cookies, no tracking, no analytics.** The embed loads static files only.
- **Clipboard access** — The `allow="clipboard-write"` attribute lets visitors copy email text. Remove it if your security policy prohibits this.

### Browser storage partitioning

Modern browsers partition `localStorage` by embedding origin. This means:
- A visitor's profiles on **your site** are separate from their profiles on **another site** using the same embed, and separate from the main DataPurge site.
- This is a privacy feature — their data doesn't leak across contexts.

### Safari / third-party storage

Safari blocks third-party storage by default. If the iframe embed has issues in Safari, the embed displays a fallback link to open DataPurge directly. The self-hosted option avoids this entirely since it runs on your domain.
